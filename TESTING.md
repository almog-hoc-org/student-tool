מסמך זה מתאר את תהליך הבדיקות המומלץ לפורטל "הדרך לדירה". הבדיקות מחולקות לכמה קטגוריות: בדיקות יחידה, בדיקות אינטגרציה, בדיקות ויזואליות, ו-edge cases.

## 4. בדיקות Edge Cases

### 4.1 קלטים קיצוניים

**מספרים גדולים מאוד**
```
- מחיר: 10,000,000 ₪
- הכנסה: 100,000 ₪
→ לא קורס, עובד כהלכה
```

**ערכים אפס**
```
- כל השדות: 0
→ תוצאות סבירות, לא קורס
```

**שדות ריקים**
```
- לחץ "חשב" עם שדות ריקים
→ הודעת שגיאה בעברית, לא קורס
```

**מספרים שליליים**
```
- הזן -1000
→ validation error: "הערך לא יכול להיות שלילי"
```

**טקסט בשדה מספר**
```
- הזן "אבג"
→ validation error: "אנא הזן מספר תקין"
```

### 4.2 תרחישים לא רגילים

**משכנתא ללא מסלולים**
```
- מחק את כל המסלולים
→ לא מאפשר מחיקה של מסלול אחרון
```

**ריבית 0%**
```
- ריבית: 0
→ חישוב נכון ללא חלוקה באפס
```

**תקופה קצרה מאוד**
```
- 1 שנה
→ תשלום חודשי גבוה, אבל נכון
```

**תקופה ארוכה מאוד**
```
- 50 שנים
→ תשלום חודשי נמוך, אבל נכון
```

## 8. איך להריץ בדיקות

### בדיקה ידנית מהירה (5 דקות)
```bash
1. npm run dev
2. פתח בדפדפן
3. נווט בין כל הדפים
4. מלא טופס אחד בכל דף
5. בדוק במובייל (F12 → device toolbar)
```

### בדיקה מקיפה (30 דקות)
```bash
1. עבור על כל מקרה מבדיקות יחידה
2. הרץ שני תרחישי אינטגרציה
3. בדוק ויזואלית ב-3 מסכים
4. נסה 5 edge cases
5. בדוק נגישות עם מקלדת
```

### בדיקת רגרסיה (לפני deploy)
```bash
1. בדיקה ידנית מהירה
2. בדוק dark mode
3. נסה שלושה תרחישים שונים
4. וודא שאין שגיאות בקונסול
5. בדוק production build
```
